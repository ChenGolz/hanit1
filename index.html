GitHub Pages-ready

Use face search on a static GitHub Pages site.

This version moves the matching into the browser. Visitors can upload a photo and search your published face library without a Python server.

GitHub Pages can host the site itself, but it cannot run your FastAPI backend or store shared public uploads. For a shared live database, you’ll still want real hosting later.

What works right now

• Public visitors can upload one image and search your published library in their browser.
• You can build test entries locally from the Build library page.
• You can export JSON and publish it in data/library.json for everyone to use.
• No backend bill for the static demo.

What still needs real hosting later

• Shared uploads into one central DB
• Private admin accounts
• Long-term storage for submitted images
• Heavy background processing for videos or bulk imports
Step 1

Create your face library

Open Build library, upload sample photos for each person, and export a JSON file.
Step 2

Publish the library

Commit your exported data into data/library.json and add thumbnail images to the repo.
Step 3

Share the search page

Send people to /search.html so they can upload one photo and get ranked matches.
